Spatial Analysis

Geoprocessing tools powered by Turf.js and PostGIS.

⬡
Buffer Analysis
Creates a buffer zone around the map centre. Counts Points of Interest (POIs) inside the zone — useful for measuring service coverage.
⟷
Measure Distance
Click two points on the map to calculate the straight-line (geodetic) distance between them. Useful for accessibility analysis.
◎
Nearest POI Finder
Click anywhere on the map. The system finds the closest Point of Interest and draws a line to it — shows travel distance.

Query Tools

Click features or search the dataset by category.

🖱
Identify Feature
Activates identify mode. Click any shape on the map to see its full attribute information (name, type, ID, etc.) from the database.
⚡
GeoServer GetFeatureInfo
Queries the GeoServer WMS directly. Click a point to get feature info from the published WMS layer — this uses the GeoServer REST API.
⬡
Search by Category
Filters loaded POI features by type — e.g. find all churches, museums or hospitals loaded from GeoServer WFS.

About

Salzburg Urban WebGIS — Final Project, Z_GIS PLUS University.

Project Aim

To develop an interactive WebGIS interface supporting urban management decision-making in Salzburg, Austria — focusing on spatial analysis, data visualisation, and accessibility tools.

Data Layers

All layers are served from GeoServer as WMS (for rendering) and WFS (for vector data). Source data: OpenStreetMap Salzburg extract.

GeoServer Connection
WMS: http://localhost:8080/geoserver/salzburg/wms
WFS: http://localhost:8080/geoserver/salzburg/ows
